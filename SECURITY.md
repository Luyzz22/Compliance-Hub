# Security Policy

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue, discussion, pull request, or support channel.

Report them privately through the security contact published at
[`https://complywithai.de/.well-known/security.txt`](https://complywithai.de/.well-known/security.txt).
Include affected versions, reproduction steps, impact, and any suggested mitigation. Do not include
personal data, production secrets, customer content, or destructive proof-of-concept payloads.

The operator must configure and monitor `COMPLIANCEHUB_SECURITY_CONTACT` before any production
release. The repository intentionally does not promise a response SLA until the operator has approved
and staffed an incident-response process.

## Supported versions

No production version is currently declared supported. Until an operator signs off the release
evidence described in `docs/enterprise-readiness-20260714.md`, this repository must be treated as
pre-production software.

## Coordinated disclosure

Please allow the maintainer reasonable time to validate and remediate a report before disclosure.
Handling, retention, access, and notification obligations for reports are governed by the operator's
approved incident-response and privacy procedures.
